Задание 3.2
Описать подробный пошаговый алгоритм, создания Пользователя на стороне бэк сервиса при вызове, получившегося в Задачи 3.1 метода. Алгоритм должен подробно описывать какие проверки требуется выполнять  бэк сервису после получения запроса на создание нового Пользователя (при нажатии кнопки “Register”),  прежде чем создать Пользователя в БД. Описание может быть в виде текста или диаграммы (например, activity)/блок-схемы.
Алгоритм создания пользователя при регистрации
1.	Получение запроса
1.	Получить JSON-данные из тела POST-запроса.
2.	Распарсить входные параметры:
firstName,
lastName,
userName,
password,
captchaToken
2.	Проверка обязательных полей
1.	Проверить наличие всех обязательных полей.
2.	Если какое-либо поле отсутствует — вернуть ошибку 400 (Bad Request).
3.	Валидация формата данных
1.	Проверить
firstName и lastName:
1.	Длина от 3 до 50 символов.
2.	Содержит только буквы и пробелы.
2.	Проверить
userName:
1.	Длина от 3 до 50 символов.
2.	Допустимы только буквы и цифры.
3.	Проверить
password:
1.	Длина от 6 до 20 символов.
2.	Содержит минимум одну заглавную букву, одну строчную букву и одну цифру.
4.	Проверить
captchaToken:
1.	Длина примерно 20-200 символов.
2.	Не пустой.
5.	Если проверки не пройдены — вернуть ошибку 400 или 422 с описанием.
4.	Проверка CAPTCHA
1.	Отправить
captchaToken на сервис reCAPTCHA для валидации.
2.	Если CAPTCHA не прошла — вернуть ошибку 400 или 422.
5.	Проверка уникальности пользователя
1.	Выполнить запрос к базе данных, чтобы проверить, существует ли уже пользователь с таким
username 
Если пользователь с таким userName уже есть — вернуть ошибку 409 (Conflict).
6.	Хеширование пароля
1.	Выполнить хеширование пароля с использованием безопасного алгоритма (например, bcrypt).
7.	Создание записи пользователя в базе данных
1.	Создать новую запись с данными:
1.	firstName,
lastName,
userName, 
хешированный пароль, 
статус (например,ACTIVE), 
и любые другие необходимые поля.
2.	Получить UserID (UUID) созданного пользователя.
